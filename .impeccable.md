# Impeccable Design Context — Aether

## Design Context

### Users
AI product managers who juggle files, conversation memory, and LLM collaboration daily. They are detail-oriented professionals who value structure and clarity. They use Aether as their cockpit — a calm, organized workspace where everything is one click away. MVP is self-use; architecture targets commercial release.

### Brand Personality
**Restrained, Refined, Effortless** (克制、精致、从容)

- **Restrained**: Every element earns its place. No decoration for decoration's sake. Whitespace is a design element, not empty space.
- **Refined**: Precision in spacing, typography weight, and micro-interactions. The difference between good and great lives in 1px adjustments.
- **Effortless**: Complex functionality presented simply. The user never feels overwhelmed — power is discoverable, not exposed.

**Emotional target**: "This tool understands me, feels comfortable, everything is under control." Professional but never cold. Minimal but every feature within reach.

### Aesthetic Direction

**Visual tone**: Nordic monochrome minimalism. Near-black + pure white + one low-saturation accent. Hierarchy built through font weight, spacing, and subtle value shifts — not color variety.

**Color philosophy**:
- **Base**: True black (#0A0A0A) and pure white (#FFFFFF) as anchors
- **Grays**: A carefully graduated neutral gray scale for layering surfaces, text, and borders
- **Accent**: A single, low-saturation muted tone — cold slate-blue or ink-green. Must be subtle enough to feel like a whisper, not a shout. Never bright or saturated.
- **Status colors**: Desaturated, muted variants of green/amber/red that harmonize with the monochrome palette

**Light + Dark mode**: Both designed simultaneously as first-class citizens. Dark mode is not an afterthought — it should feel native, not inverted.

**Typography**:
- **English/UI**: Space Grotesk Variable — geometric sans-serif with personality, distinct from generic Inter/Roboto
- **Chinese**: Noto Sans SC Variable — clean, modern CJK with excellent weight range
- **Code**: JetBrains Mono Variable — industry standard monospace
- Hierarchy through weight contrast (400 body / 500 labels / 600 headings / 700 display) and deliberate size jumps
- Type scale: caption (11px) → xs (12px) → sm (14px) → base (16px) → 2xl (24px) — clear tiers with no muddy middle
- Generous line-height for body (1.7-1.8), tight for headings (1.2-1.4)

**Illustration style**: Line-art, hand-drawn feel (as in the reference UI). No photographs, no filled/solid icon sets, no stock imagery. When illustration is needed, think editorial sketch — sparse, human, slightly imperfect.

**Spacing**: 8px grid, generous whitespace. Content breathes. Padding is never stingy.

**Corner radius**: Consistent 8px (rounded-lg) for cards and containers, 6px for inputs and small buttons, full-round for pills/tags.

**References**:
- **Notion**: Structural clarity, black-and-white dominance, typography-driven hierarchy, generous whitespace
- **Linear**: Refined interactions, attention to micro-detail, elegant motion
- **Claude**: Warmth and human-centricity (but NOT its terracotta palette)
- **Reference UI** (uploaded): Near-monochrome, font-weight-driven hierarchy, line-art illustrations, card-based content with soft shadows

**Anti-references** (DO NOT):
- Cyberpunk / neon / dark-with-glow aesthetics
- High-saturation multi-color palettes
- Cheap utility-tool websites
- Purple gradient + Inter font "AI slop" aesthetic
- Claude's terracotta/warm-earth palette (Aether needs its own identity)
- Solid/filled icon sets, emoji icons (MVP exception, migrate to line icons)
- Bounce, jelly, or elastic easing on any animation

### Design Principles

1. **Monochrome first, color by exception**
   Almost everything is black, white, and gray. The accent color appears only for primary actions, active states, and key focal points. If you're reaching for a color, ask: can this be solved with weight, size, or spacing instead?

2. **Typography is the design**
   Weight contrast (regular vs semibold vs bold) and size steps create all the hierarchy you need. A well-set heading with proper spacing communicates more than any colored badge.

3. **Whitespace is intentional**
   Every margin and padding is deliberate. Content has room to breathe. Dense information is organized through structure (sections, dividers, indentation), never crammed.

4. **Motion is earned**
   Animations serve spatial continuity and interaction feedback only. Subtle ease-out curves (150-250ms). Hover states may include a gentle 1-2px lift or opacity shift. Expand/collapse slides smoothly. No bounce, no jelly, no spring physics. Always respect `prefers-reduced-motion`.

5. **Quiet confidence**
   The interface doesn't shout. Success is a small checkmark, not a celebration. Errors are clear but calm. Empty states guide without patronizing. The tool feels like a trusted colleague — competent, reliable, understated.

### Animation Guidelines

| Interaction | Duration | Easing | Effect |
|---|---|---|---|
| Hover background | 150ms | ease | Color transition only |
| Hover lift | 200ms | ease-out | translateY(-1px) + subtle shadow |
| Focus ring | 150ms | ease | Border/ring appear |
| Panel expand/collapse | 250ms | ease-in-out | Height + opacity |
| Sidebar collapse | 300ms | cubic-bezier(0.4, 0, 0.2, 1) | Width transition |
| Toast enter | 250ms | cubic-bezier(0, 0, 0.2, 1) | translateY(8px) + opacity |
| Toast exit | 150ms | ease-in | Opacity fade |
| Dropdown open | 200ms | cubic-bezier(0, 0, 0.2, 1) | Scale(0.97→1) + opacity |
| Page/tab switch | 150ms | ease | Opacity crossfade |

**Reduced motion**: When `prefers-reduced-motion: reduce`, replace all transforms and slides with simple opacity transitions (150ms). Keep functional state changes instant.

### Component Style Guide

**Buttons**:
- Primary: `bg-accent text-white` — used sparingly (1-2 per view max)
- Secondary: `border border-border text-primary` — most common
- Ghost: `text-secondary hover:text-primary hover:bg-hover` — toolbar actions
- All buttons: 8px radius, 32-36px height, font-weight 500

**Inputs**:
- 36px height, 6px radius, 1px border
- Focus: border transitions to `text-secondary` (not accent — stays monochrome)
- Placeholder in `text-muted`

**Cards/Panels**:
- 8px radius, 1px `border` border
- No heavy shadows in light mode — elevation via border only
- Dark mode: may add `shadow-sm` for subtle depth

**Tags/Badges**:
- Full-round (pill shape), small (22-24px height)
- Active: `bg-accent-subtle text-accent border-accent`
- Inactive: `bg-canvas text-secondary border-border`

**Dividers**:
- 1px `border-border` — never heavy, never dark
- Use sparingly — prefer whitespace over lines

**Icons** (future migration from emoji):
- 1.5px stroke weight, rounded caps/joins
- 16-20px standard size
- `currentColor` for automatic theme adaptation
- Style: Lucide or similar minimal line-icon set

### Accessibility Requirements

- WCAG AA compliance minimum for all text
- `text-muted` must achieve 3:1 contrast on its background (for large text / UI components) or 4.5:1 (for body text)
- All interactive elements keyboard-accessible with visible focus indicators
- Focus ring: 2px offset, uses `text-secondary` color (not accent — stays monochrome)
- `aria-label` on all icon-only buttons
- `prefers-reduced-motion` respected in all animations
- `prefers-color-scheme` detected for automatic theme selection
- Color is never the sole indicator of state (always paired with icon, text, or shape)
